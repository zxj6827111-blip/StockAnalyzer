# 自学习体系优化优先级清单 v1.0

## 结论

当前这套系统的夜间/周末自学习框架方向正确，但离“可长期稳定演进的正式体系”还有几个关键缺口。

现阶段最值得先做的 5 个点，按优先级排序如下：

## P0

### P0-1：补齐真实新闻采集与 M7 入库
- 目标：让系统不再主要依赖本地静态新闻文件或 `market_proxy` 伪新闻
- 现状：`M7` 已有新闻学习框架，但真实新闻输入仍不稳定
- 价值：直接提升盘前简报、盘中事件识别、夜间新闻学习质量
- 建议动作：
  - 增加真实新闻采集器
  - 统一写入 `artifacts/evolution/inputs/m7_news_latest.jsonl`
  - 白天盘前/午间可复用，晚上自学习可直接读取

### P0-2：打开周末“重型升级”能力
- 目标：让周末真正承担深度学习/大样本评估/记忆整理职责
- 现状：`idle_queue` 机制存在，但默认未自动启用
- 价值：把日常小修小补和周末深度升级分层
- 建议动作：
  - 在模拟盘阶段先启用周末 `idle_queue`
  - 周末专门做长样本复盘、候选模型 shadow 评估、M3/M8 深层清理

## P1

### P1-1：把 1 分钟 / 5 分钟复盘数据接入夜间学习
- 目标：让系统不只“会盯盘中分钟线”，还“会学习分钟线”
- 现状：白天实时监控已支持 1m/5m，夜间学习仍以日线主导
- 价值：提升盘中突破、冲高回落、假强势识别
- 建议动作：
  - 在夜间构建分钟级摘要特征
  - 写入 M5/M11 或单独的 intraday replay 样本

### P1-2：建立冠军 / 挑战者 / 影子组合的正式晋升闭环
- 目标：让模型升级从“有建议”升级为“有闭环”
- 现状：已有 `champion_v7`、M8 建议、M11 影子组合，但晋升链路仍偏轻
- 价值：显著降低自动升级误伤风险
- 建议动作：
  - 候选模型先进入 challenger
  - 通过 shadow portfolio 验证
  - 再进入模拟盘观察
  - 最后才允许晋升 champion

## P2

### P2-1：扩大夜间研究样本池，白天候选池保持精简
- 目标：把“夜里研究”和“白天盯盘”彻底分开
- 现状：白天候选池已收敛到 50 只，但夜间研究样本仍可扩大
- 价值：提高热点发现能力，减少漏票
- 建议动作：
  - 夜间研究池维持 `Top500 ~ Top1000`
  - 白天监控池保持 `Top50`
  - 学习过程中发现热点股时，允许动态补池

## 推荐实施顺序

1. `P0-1` 真实新闻采集 + M7 入库
2. `P0-2` 周末重型升级启用
3. `P1-1` 分钟级复盘学习
4. `P1-2` 冠军/挑战者闭环
5. `P2-1` 夜间研究池扩容

## 当前建议

如果只先做一件事，优先做：

`真实新闻采集 + M7 入库 + AI 辅助审核开关`

这是现阶段性价比最高、对白天推送和夜间学习都同时增益的一项改造。
